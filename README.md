# ChatLogViewer

Windows 11 上の VS Code ローカル保存データを走査し、チャット履歴を一覧表示・閲覧・Markdown 出力するデスクトップアプリケーションです。

対象は標準的な VS Code 環境です。Insiders、Visual Studio、Cursor などは対象外です。

## 対応環境

- OS: Windows 11
- Python: 3.12 以上
- UI: Tkinter（標準ライブラリのみ、外部依存なし）

## 何をするアプリか

起動時に VS Code のユーザ保存領域を自動スキャンし、ローカルに保存されているチャットセッションを読み取ります。

主な用途:

- チャット履歴の一覧表示
- セッション内容の閲覧
- 原本ファイルやワークスペースへの導線確保
- Markdown への保存

## 読み取り対象

既定では次の VS Code 保存先を走査します。

- `%APPDATA%\Code\User\workspaceStorage\<workspace-hash>\chatSessions\*.jsonl`

補助的に次も参照します。

- `%APPDATA%\Code\User\workspaceStorage\<workspace-hash>\workspace.json`

`workspace.json` から、対応するワークスペースやフォルダのパスを復元します。

## 画面構成

### 左カラム

履歴一覧を表示します。表示列は以下です。

- メッセージ数
- 最終更新日時 `yyyy/mm/dd HH:MM`
- チャット先頭メッセージの冒頭

補足:

- 複数選択できます
- 列見出しクリックで並び替えできます

### 右カラム

選択した履歴の内容を Markdown 風表示で閲覧できます。

表示内容:

- 保存されている JSONL ファイルパス
- ワークスペースパス
- セッション本文

操作:

- `保存フォルダを開く`
- `Workspaceを開く`

## Markdown 出力

### 単体保存

1 件選択時は保存先ファイルを指定して Markdown を保存できます。

### 複数保存

複数選択時は保存先フォルダを選び、複数の Markdown をまとめて出力できます。

同名ファイルがある場合は、`(2)`, `(3)` のように連番を付けて重複を避けます。

## Markdown の内容

出力される Markdown には次のような情報が含まれます。

- セッションタイトル
- セッション ID
- Windows ユーザ名
- ワークスペース
- 原本 JSONL ファイルパス
- 作成日時
- 更新日時
- モデル ID
- 会話本文

ユーザ発言には送信時刻 `yyyy/mm/dd HH:MM` を付記します。

## 復元方針

VS Code の保存形式はストリーミング途中断片や内部イベントを含むため、会話本文は完全なログそのものではなく「閲覧しやすい形への復元」です。

現時点では次の方針を取っています。

- ユーザ発言は request のメッセージと timestamp を利用
- AI 側は visible な response 断片を復元
- `inlineReference` は本文リンク表現として復元
- 一部の進捗表示は `result.metadata.toolCallRounds[].response` を優先利用
- 壊れた末尾行があっても、読める範囲は継続して取り込み

## 制約

以下は既知の制約です。

- VS Code の内部保存形式は頻繁に変わる可能性があります
- AI 応答時刻は、少なくとも現在確認した保存形式では厳密には取れません
- 一部セッションでは、途中進捗文と最終回答の境界が完全には復元できない場合があります
- そのため、完全な再現よりも「破綻しにくく読みやすい復元」を優先しています

## セットアップと起動

パッケージ管理に [uv](https://docs.astral.sh/uv/) を使用しています。

### 環境準備（初回のみ）

```powershell
uv sync --no-dev
```

開発用ツール（テストフレームワーク・静的解析）は含まれません。アプリを動かすだけであればこれで十分です。

### 起動

```powershell
uv run src/main.py
```

## 保存形式への追従について

本アプリは VS Code が内部的に書き出す JSONL ファイルを独自に解析しています。VS Code 本体のアップデートにより **保存形式が予告なく変更された場合、正しく復元できなくなる可能性があります**。最新形式への自動追従は行っておらず、必要に応じて手動で対応を行う方針です。
