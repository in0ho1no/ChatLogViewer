# ローカルAIチャット履歴ビューア 仕様書

## システム概要

VS Codeおよびその拡張機能がWindowsローカルに保存しているチャット履歴を抽出し、一覧表示・閲覧・Markdown出力を行うデスクトップアプリケーション。標準的なインストール環境では「起動即スキャン」を実現しつつ、特殊な環境にも対応できる柔軟性を備える。

## 動作環境

* OS: Windows 10/11
* 言語: Python 3.x (標準ライブラリのみ使用)
* UI: Tkinter (標準GUIキット)

## パス解決・探索ロジック

アプリ起動時、以下の優先順位で解析対象ディレクトリを決定する。

### デフォルトパスの自動検知

1.  環境変数 `%APPDATA%\Code\User` をデフォルトの探索起点とする。
1.  起動時に上記ディレクトリ配下に `workspaceStorage` または `globalStorage` が存在するか確認する。
1.  存在する場合、ユーザーの操作を待たずに自動的にスキャンを開始し、履歴一覧を表示する。

### 任意パスの手動指定 (フォールバック)

以下の状況において、ユーザーが任意のディレクトリを選択できるUIを提供する。
* デフォルトパスが存在しない場合（VS Code Insiders利用時、ポータブル版利用時など）。
* ユーザーが特定のバックアップフォルダや、特定のワークスペースのみを指定したい場合。
* UI上の「フォルダを選択」ボタンから、`%APPDATA%` 外の場所も指定可能とする。

## 入力データ仕様

* 対象ディレクトリ:
    * `workspaceStorage\`
    * `globalStorage\`
* 解析対象ファイル:
    1.  `.json`: `requests` または `messages` 配列を持つチャットセッションファイル。
    2.  `.vscdb` (SQLite3): `ItemTable` 内にJSON文字列として格納されているセッションデータ。

## 機能要件

### 自動スキャンとステータス表示

* 起動時、自動探索の結果（「○件のファイルをスキャン中...」等）をUI上に表示すること。
* 有効なチャットデータが見つからない場合は、手動選択を促すメッセージを表示すること。

### セッション一覧とソート

* 左ペインにセッション一覧を表示。
* 最新順 (降順) / 古い順 (昇順) の切り替えスイッチを設ける。
* ソート基準は「ファイルの最終更新日時」または「セッション内の最新タイムスタンプ」とする。

### 詳細閲覧

* 選択したセッションの内容を右ペインに表示。
* ユーザーとAIの発言を色分け、またはラベル（👤 / 🤖）で区別して表示する。
* コードブロック等の等幅フォント表示への配慮。

### Markdownエクスポート

* 閲覧中のセッションを `.md` 形式で保存。
* ファイル名はセッション日時をベースに自動生成し、保存ダイアログで変更可能とする。

## 非機能要件

* ゼロ・コンフィギュレーション: インストール済みのVS Codeに対しては、設定不要で即座にデータにアクセスできること。
* プライバシーの担保: 外部通信を一切行わず、すべての処理をローカルメモリ内と標準出力のみで完結させること。
* メンテナンス性: Python標準ライブラリ（`os`, `json`, `sqlite3`, `tkinter`）のみを用いることで、将来的な環境変化に強く、`uv run` 等で即座に実行可能なポータビリティを維持すること。
